- D:\JannatBooking\hotels_backend\controllers\zadcontroller.js
  - Main Zad public website controller.
  - Main manager/editor controller for the Zad website document.
  - Owner email constant:
    process.env.ZAD_OWNER_EMAIL || "mrgamal@xhoteltest.com"
  - Important behavior:
    - Public hotel endpoints are fenced to hotels connected to the Zad owner email.
    - Public hotel responses strip private agent room override data.
    - Website document updates are allowed only to the Zad website manager email or configured platform super admin.
    - Public list endpoints use short in-memory cache headers and compact response shapes.

- D:\JannatBooking\hotels_backend\routes\zad.js

  - Uses requireSignin/isAuth for saving website document.
  - Uses userById route param for authenticated editor saves.

Public API Endpoints
All endpoints below are mounted under /api.

- GET /api/zad-website-document
  - Controller:
    listZadWebsiteDocuments
  - Used by:
    D:\JannatBooking\zad_ssr\lib\api.js
    D:\JannatBooking\hotels_frontend\src\HotelModule\apiAdmin.js
  - Returns the saved Zad website document or defaults if none exists.

- GET /api/zad/scope-health
  - Controller:
    getZadScopeHealth
  - Purpose:
    Quick diagnostic for owner email resolution and hotel scope count.

- GET /api/zad/active-hotels
  - Controller:
    listOfAllActiveZadHotels
  - Used by the home page featured hotels.

- GET /api/zad/active-hotel-list
  - Controller:
    getZadListOfHotels
  - Used by:
    hotel listing
    support widget hotel choices
    search/filter pages

- GET /api/zad/distinct-rooms
  - Controller:
    distinctZadRoomTypes
  - Used by the room search form.

- GET /api/zad/single-hotel/:hotelSlug
  - Controller:
    getZadHotelFromSlug
  - Used by:
    D:\JannatBooking\zad_ssr\app\single-hotel\[slug]\page.js

- GET /api/zad/room-query-list/:query
  - Controller:
    gettingZadRoomListFromQuery
  - Query format:
    startDate_endDate_roomType_adults_children_destination
  - Used by:
    D:\JannatBooking\zad_ssr\app\rooms\page.js

- GET /api/zad/hotels/active-with-deals
  - Controller:
    listOfAllActiveZadHotelsMonthlyAndOffers
  - Used by:
    D:\JannatBooking\zad_ssr\app\zad-offers-monthly-reservations\page.js

Hotel Scope Logic
- Zad owner email:
  mrgamal@xhoteltest.com by default.
- Controller resolves the User document by email.
- Controller collects hotel ids from:
  - hotelIdWork
  - hotelIdsWork
  - hotelsToSupport
  - hotelIdsOwner
  - HotelDetails documents where belongsTo is the owner id or belongsToId
- Public hotel filter requires:
  - _id in resolved Zad hotel ids
  - activateHotel: true
  - xHotelProActive not false
  - hotel photos
  - valid location coordinates
  - active public rooms with base price and room photos for most list endpoints

Important Support Widget Rule
- The Zad SSR support widget must not use the old Jannat/global support hotel id.
- It requires a real Zad hotel selection from /api/zad/active-hotel-list.
- It sends supportScope="hotel".
- It sends hotelId and ownerId from the selected Zad hotel.
